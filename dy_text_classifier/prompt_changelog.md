## 变更历史

### V3.2.5 (2026-03-12) - 已发布 ✅

#### 【重要修复】移除错误的文件缓存机制，修复视频信息固定问题

1. **问题反馈 (BUG-001)**
   - 现象：提示词中的"视频信息"（作者、文案）与实际视频不符
   - 示例：视频A（作者@荷映夏，文案"豆包起的新网名"）收到了视频B（作者@科技茶馆TEA，文案"谷歌不卷AI"）的提示词
   - 根因：文件缓存机制设计错误，缓存了填充后的提示词，但缓存Key不包含video_data，导致不同视频共享错误缓存

2. **影响评估**
   - 严重程度：🔴 严重
   - 影响范围：所有使用缓存的评论生成
   - 后果：模型接收矛盾信息，生成评论偏离实际视频内容

3. **修复方案**
   - 移除文件缓存机制（`获取评论助手提示词` 方法）
   - 直接调用 `_生成增强版评论提示词` 生成提示词
   - 确认：该方法仅做本地字符串替换，无LLM API调用
   - 性能：单次生成 < 1ms，无需缓存优化

4. **代码变更详情**
   - 删除：Hash计算逻辑
   - 删除：缓存检查逻辑 (`_load_from_cache`)
   - 删除：缓存保存逻辑 (`_save_to_cache`)
   - 保留：`_生成增强版评论提示词` 方法（本地字符串替换）
   - 清理：删除 `prompt_cache/` 目录下所有旧缓存文件

5. **验证结果**
   ```
   ✅ prompt_A 包含 @科技茶馆TEA: True
   ✅ prompt_A 包含 谷歌不卷AI: True
   ✅ prompt_B 包含 @荷映夏: True
   ✅ prompt_B 包含 豆包起的新网名: True
   ✅ 两者不相同: True
   ```

#### 模板变更
- 无模板内容变更（仅代码逻辑修复）

#### 代码变更
- `PROMPT_TEMPLATE_VERSION`: 3.2.4 → 3.2.5
- `获取评论助手提示词`: 移除缓存逻辑，直接生成
- `prompt_cache/`: 清理所有旧缓存

### V3.2.4 (2026-03-09) - 上一版本

#### 进一步降低"很好奇"句式使用概率
1. **问题反馈**
   - 仍有约25%概率出现"很好奇"句式
   - 示例："看到"国士无双"就知道这局牌局肯定不简单，很好奇你是怎么一步步凑齐这副牌的？"

2. **修复方案**
   - 使用比例限制：30% → 10%
   - 强化约束描述："严格限制"、"优先使用其他句式"
   - 策略A示例更新：明确标注"很好奇"为差示例
   - 新增好示例：直接疑问句式替代"很好奇"

3. **示例对比**
   - 差示例（很好奇）："很好奇你是怎么一步步凑齐这副牌的？"
   - 好示例（直接疑问）："这副国士无双，听牌的时候是怎么决策的？"

#### 模板变更
- 句式多样性约束："不超过30%" → "不得超过10%"
- 策略A示例：明确标注"很好奇"为差示例
- 格式约束：强调"优先使用直接疑问、观点陈述、经验分享"

#### 代码变更
- `PROMPT_TEMPLATE_VERSION`: 3.2.3 → 3.2.4
- `_提示词模板基础内容`: 更新句式多样性约束和示例

### V3.2.3 (2026-03-09) - 已发布 ✅

#### 避免泛泛问题，要求专业深度
1. **问题修复**
   - 问题：评论过于泛泛，如"Blender建模的练习过程中，你是如何平衡效率和细节的？"
   - 缺乏专业术语和具体实例

2. **修复方案**
   - 字数上限扩展：100字 → 150字（容纳专业内容）
   - 策略A示例更新：区分"差示例"和"好示例"
   - 新增约束：
     - 避免"如何平衡..."、"有什么技巧"、"怎么做到"等泛泛问题
     - 必须使用专业术语（如"拓扑优化"、"烘焙贴图"、"法线贴图"、"LOD"）
     - 必须带上具体实例或技术方案

3. **好示例 vs 差示例**
   - 差："Blender建模的练习过程中，你是如何平衡效率和细节的？"（泛泛）
   - 好："Blender的雕刻模式下，你是用Dyntopo还是Multires来做硬表面细节的？"（具体专业）
   - 好："拓扑优化时，如何控制三角面数量和布线走向以满足动画形变需求？"（专业术语）

#### 模板变更
- 字数约束：30-100字 → 30-150字
- 策略A：更新示例，强调具体专业
- 格式约束：新增"避免泛泛问题"、"使用专业术语"要求

#### 代码变更
- `PROMPT_TEMPLATE_VERSION`: 3.2.2 → 3.2.3
- `_提示词模板基础内容`: 更新策略A示例，增加专业术语要求

### V3.2.1 (2026-03-09) - 已发布 ✅

#### 风控安全修复
1. **禁止@任何人**
   - 问题：评论中出现"@ps平面设计（ai, cdr）"，存在风控风险
   - 修复：添加明确约束"评论中严禁@作者或任何其他用户"
   - 原因：平台对@行为有风控限制，频繁@可能导致账号异常

#### 模板变更
- 格式约束：将"@作者：约30%概率"改为"**禁止@任何人**"
- 强调：出于风控考虑，严禁@用户名、@作者、@视频主等

#### 代码变更
- `PROMPT_TEMPLATE_VERSION`: 3.2.0 → 3.2.1
- `_提示词模板基础内容`: 更新@约束为禁止
- `_生成增强版评论提示词`: 注释更新，作者仅用于参考

### V3.2.0 (2026-03-09) - 已发布 ✅

#### 问题修复
1. **时间错误修复**
   - 问题：评论中出现"2025年的渲染技术"，实际已是2026年
   - 修复：添加当前时间变量 `{{当前时间}}`，禁止提及具体年份
   - 实现：`datetime.now().strftime("%Y年%m月")` → "2026年03月"

2. **策略多样性优化**
   - 原问题：所有评论都用策略A（专业钩子问题），过于单一
   - 调整：A(30%)、B(50%)、C(15%)、D(5%)
   - 新增策略D：轻松互动型（针对轻松/娱乐内容）
   - 示例："渲染小曲一响，DNA就动了😂"

3. **语境感知增强**
   - 问题：对娱乐性视频使用过于正式的评论
   - 优化：添加视频类型判断指南
     - 技术教程（含"教程"、"技巧"）→ 专业深度
     - 娱乐休闲（含"小曲"、"日常"）→ 轻松幽默
     - 比赛参与（含"大赛"、"参赛"）→ 鼓励支持
     - 热点讨论（有明显立场）→ 观点鲜明

4. **敏感暗示移除**
   - 问题："在不使用AI的情况下"可能引发争议
   - 修复：删除AI相关暗示，避免过度专业词汇用于娱乐视频

#### 模板变更
- 添加 `{{当前时间}}` 变量
- 新增语气适配指令
- 调整策略概率分布（A30/B50/C15/D5）
- 添加视频类型判断提示
- 强化时间约束："禁止提及过时年份"

#### 代码变更
- `PROMPT_TEMPLATE_VERSION`: 3.1.0 → 3.2.0
- `_提示词模板基础内容`: 添加时间信息和语境感知
- `_生成增强版评论提示词`: 添加当前时间变量
